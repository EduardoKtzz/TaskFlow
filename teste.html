Estrutura de um projeto Nuxt

Nuxt organiza seu projeto em pastas com significado próprio. Cada pasta faz uma coisa específica — e o framework entende automaticamente.

ESTRUTURA DE PASTAS O projeto por dentro

Ao contrário do React puro, no Nuxt você não configura rotas manualmente. A pasta /pages vira rotas automaticamente.

meu-projeto/
  pages/           ← rotas automáticas
    index.vue       ← rota /
    sobre.vue       ← rota /sobre
    [id].vue        ← rota /123 (dinâmica)
  components/      ← componentes reutilizáveis
    Botao.vue
    Card.vue
  composables/     ← lógica reutilizável (como hooks React)
    useContador.ts
  layouts/         ← templates de layout (header/footer)
  server/          ← API routes backend (como Next.js API)
    api/
      usuarios.get.ts ← GET /api/usuarios
  nuxt.config.ts   ← configuração central
  app.vue         ← componente raiz
ANALOGIA Nuxt é para Vue o que Next.js é para React
Next.js (React)
/pages/index.tsx → rota /
/pages/[id].tsx → rota dinâmica
useHook() → lógica reutilizável
/pages/api/ → backend
getServerSideProps → SSR
Nuxt (Vue)
/pages/index.vue → rota /
/pages/[id].vue → rota dinâmica
useComposable() → lógica reutilizável
/server/api/ → backend
useFetch() → SSR data fetching

Anatomia de um componente .vue

Todo arquivo .vue tem três blocos. Template (HTML), Script (lógica) e Style (CSS). Simples assim.

ESTRUTURA Os 3 blocos de um .vue
<!-- MeuComponente.vue --> <template> <!-- 1. HTML do componente --> <div> <h1>{{ titulo }}</h1> <button @click="incrementar">Clicou {{ vezes }}x</button> </div> </template> <script setup lang="ts"> // 2. Lógica TypeScript import { ref, computed } from 'vue' const titulo = 'Meu componente' // prop simples const vezes = ref(0) // estado reativo (= useState) const incrementar = () => { vezes.value++ // .value é obrigatório no script } </script> <style scoped> /* 3. CSS isolado nesse componente */ h1 { color: blue; } </style>

Diretivas Vue

Diretivas são atributos especiais que começam com v-. Elas conectam o HTML à sua lógica JavaScript.

v-model Two-way binding — input ↔ variável

O que você digita no input atualiza a variável em tempo real. E a variável atualiza o input. Mão dupla.

<input v-model="nome" placeholder="Digite algo" /> <p>Olá, {{ nome }}!</p> // No script: const nome = ref('')
// DEMO AO VIVO — v-model
Olá, {{ nomeInput }}! aguardando...
v-for Renderizar listas

Equivalente ao .map() do React. Adicione e remova itens — a lista atualiza sozinha.

<div v-for="(tarefa, index) in tarefas" :key="index"> {{ tarefa.texto }} </div>

v-if / v-else Renderização condicional

Mostra ou esconde elementos baseado em uma condição.

<div v-if="logado">Bem-vindo!</div> <div v-else>Faça login.</div>

// 04 — Reutilização

Composables

Composables são funções que encapsulam lógica com estado — o equivalente dos hooks do React. Prefixo use por convenção.

useContador.ts Criando um composable

Você extrai lógica repetida para um arquivo separado e importa onde precisar. Qualquer componente pode usar o mesmo useContador().

// composables/useContador.ts import { ref, computed } from 'vue' export function useContador(inicial = 0) { const count = ref(inicial) const dobro = computed(() => count.value * 2) function incrementar() { count.value++ } function decrementar() { count.value-- } function resetar() { count.value = inicial } return { count, dobro, incrementar, decrementar, resetar } } // Usando no componente: const { count, dobro, incrementar, decrementar, resetar } = useContador(10)
// DEMO — composable useContador() em ação
{{ contador }}

useFetch() Buscar dados do servidor — built-in do Nuxt

O Nuxt tem composables prontos. useFetch() busca dados e funciona no SSR automaticamente.

// pages/usuarios.vue <script setup> // useFetch é automático no Nuxt — sem import! const { data: usuarios, pending, error } = await useFetch('/api/usuarios') </script> <template> <div v-if="pending">Carregando...</div> <div v-else-if="error">Erro ao carregar</div> <ul v-else> <li v-for="u in usuarios" :key="u.id">{{ u.nome }}</li> </ul> </template>

// 06 — Ciclo de vida

Lifecycle hooks

Hooks que disparam em momentos específicos da vida do componente — montagem, atualização, desmontagem.

HOOKS Vue vs React
React
useEffect(() => {}, [])
→ ao montar
useEffect(() => {}, [x])
→ quando x muda
return () => cleanup()
→ ao desmontar
Vue
onMounted(() => {})
→ ao montar
watch(x, () => {})
→ quando x muda
onUnmounted(() => {})
→ ao desmontar
CÓDIGO Exemplo prático
import { onMounted, onUnmounted, watch, ref } from 'vue' const dados = ref([]) const busca = ref('') // Executa quando o componente é montado na tela onMounted(async () => { dados.value = await buscarDados() }) // Observa mudança na variável "busca" watch(busca, (novoValor) => { console.log('busca mudou para:', novoValor) }) // Limpeza ao sair da página onUnmounted(() => { clearInterval(meuInterval) })

O que falar amanhã

Resumo das respostas certas para as perguntas mais prováveis sobre Vue e Nuxt.

PERGUNTA "Você tem experiência com Vue?"
"Minha experiência de frontend é com React, mas estudei Vue recentemente e os conceitos são muito similares. ref() é o useState, v-for é o .map(), @click é o onClick, e Nuxt está para Vue assim como Next.js está para React. Me sinto confortável para aprender o ecossistema Vue com o time."
PERGUNTA "Qual a diferença entre ref() e reactive()?"
// ref() — para valores simples (string, number, boolean) const nome = ref('Eduardo') console.log(nome.value) // precisa de .value no script // reactive() — para objetos const usuario = reactive({ nome: 'Eduardo', idade: 22 }) console.log(usuario.nome) // acesso direto, sem .value
PERGUNTA "O que é um composable?"
"É o equivalente dos hooks customizados do React. Uma função que encapsula lógica com estado e pode ser reutilizada em qualquer componente. Por convenção começa com use — como useContador(), useFetch(), useAuth()."
PERGUNTA "O que o Nuxt adiciona ao Vue?"
"Nuxt é um framework fullstack em cima do Vue. Adiciona roteamento automático por pasta, SSR e SSG nativo, composables built-in como useFetch() e useState(), rotas de API no servidor, e muito menos configuração. É o Next.js do Vue."
